# Fix: ROS2 nodes not discoverable on external WiFi

## Context

When the Jetson Orin boots, the ROSMASTER hotspot (priority 2) activates first. Autostart scripts launch `micro_ros_agent`, which binds DDS unicast sockets to `192.168.8.88` (hotspot IP). If the network later switches to external WiFi (MHL → `192.168.50.103`), those sockets are stale — DDS discovery fails silently.

**Multicast works fine on MHL.** The problem is FastDDS binding to a specific interface IP instead of all interfaces.

## Solution: FastDDS XML profile binding to 0.0.0.0

Create a FastDDS XML profile that forces all transports to listen on `0.0.0.0` (all interfaces). Set `FASTRTPS_DEFAULT_PROFILES_FILE` in all startup scripts so every ROS2 node uses it.

### Files to create/modify on robot (via SSH)

#### 1. Create `/home/jetson/fastdds_all_interfaces.xml`

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<dds xmlns="http://www.eprosima.com/XMLSchemas/fastRTPS_Profiles">
    <transport_descriptors>
        <transport_descriptor>
            <transport_id>udp_all</transport_id>
            <type>UDPv4</type>
            <interfaceWhiteList>
                <address>0.0.0.0</address>
            </interfaceWhiteList>
        </transport_descriptor>
    </transport_descriptors>
    <profiles>
        <participant profile_name="default_participant" is_default_profile="true">
            <rtps>
                <userTransports>
                    <transport_id>udp_all</transport_id>
                </userTransports>
                <useBuiltinTransports>false</useBuiltinTransports>
            </rtps>
        </participant>
    </profiles>
</dds>
```

#### 2. Modify `/home/jetson/.bashrc`
Add before the ROS2 source lines:
```bash
export FASTRTPS_DEFAULT_PROFILES_FILE=/home/jetson/fastdds_all_interfaces.xml
```

#### 3. Modify `/home/jetson/start_agent.sh`
Add `export FASTRTPS_DEFAULT_PROFILES_FILE=/home/jetson/fastdds_all_interfaces.xml` before the `ros2 run` command.

#### 4. Modify `/home/jetson/joy_control/joy.sh`
Add the same export before sourcing ROS2.

#### 5. Modify `/home/jetson/.config/autostart/check_sensor.desktop`
Add `export FASTRTPS_DEFAULT_PROFILES_FILE=/home/jetson/fastdds_all_interfaces.xml &&` to the Exec line.

### Steps

1. Create the FastDDS XML profile
2. Edit the 4 startup files to export `FASTRTPS_DEFAULT_PROFILES_FILE`
3. Kill stale processes and restart to test immediately
4. Reboot to verify it persists

## Immediate test (before reboot)

1. Kill stale processes: `killall micro_ros_agent; kill 194641`
2. Export the profile and restart micro_ros_agent manually
3. Check `ros2 topic list` with `ROS_DOMAIN_ID=30`

## Verification

1. Reboot robot on MHL WiFi
2. SSH in: `source /opt/ros/humble/setup.bash && export ROS_DOMAIN_ID=30 && export FASTRTPS_DEFAULT_PROFILES_FILE=/home/jetson/fastdds_all_interfaces.xml && ros2 topic list`
3. Confirm sensor topics appear (`/scan0`, `/scan1`, `/imu/data_raw`)
4. Verify `ss -ulnp | grep micro_ros` shows `0.0.0.0` bindings (not `192.168.8.88`)
5. Switch to ROSMASTER hotspot and verify topics still work
